# F1 赛季预测对比系统（MVP）页面与交互设计

## 全局

- 顶部导航：比赛、排行榜、（管理员入口）、右侧用户菜单
- 视觉：中性灰 + 单一强调色，移动端优先

## 页面 1：登录/注册（`/login`）

组件：

- 邮箱输入、密码输入
- 登录按钮 / 注册按钮
- 状态提示（错误、加载）

行为：

- 成功后跳转 `/`

## 页面 2：比赛列表（`/`）

内容：

- 当前赛季（year）
- 按 round 分组的卡片列表
  - 每张卡展示：`grand_prix_name`、`session_type`、`event_date`
  - 状态：距离截止时间倒计时/已锁定/已结束

行为：

- 点击进入比赛详情 `/events/:id`

## 页面 3：预测页（`/events/:id`）

内容：

- Session 信息 + 截止时间
- 预测表单（Top5）
  - 5 个位置，每个位置一个 Driver 下拉
  - 已选车手在其他下拉中禁用
- 提交按钮

校验：

- 必须选满 5 个
- 不允许重复
- 超过截止时间：表单禁用并显示“已锁定”

## 页面 4：排行榜与折线图（`/leaderboard`）

内容：

- 排行榜表格：排名、昵称、总分、已完赛场次
- 折线图：每位用户一条线
  - X：round（同一 round 多 session 时按 event 排序累加）
  - Y：累计分

交互：

- 用户筛选（显示/隐藏某用户曲线）

## 页面 5：管理员首页（`/admin`）

- 卡片入口：车手、赛季、比赛、结果录入

权限：

- 非管理员访问显示无权限并引导回首页

## 页面 6：车手管理（`/admin/drivers`）

- 列表：name/team/is_active
- 新增/编辑弹窗
- 启用/停用开关

## 页面 7：赛季与比赛管理（`/admin/seasons`、`/admin/events`）

- 创建赛季（year）
- 创建 event：season、round、gp 名、session、日期、截止时间

## 页面 8：结果录入（`/admin/results`）

- 选择 event
- 录入前 5
- 保存结果
- finalize（确认弹窗）
- finalize 后显示本场已计分
